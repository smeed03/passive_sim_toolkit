# IoT Traffic Simulation Toolkit

This toolkit contains lightweight scripts to simulate IoT network activity, currently supporting:

- **MQTT telemetry / control bursts** using `paho-mqtt`
- **SSDP LAN discovery chatter** using `scapy`
- **Scenario runners** to simulate passive → active → passive behavior

---

# Requirements

## Python
Python 3.9+ recommended.

Install dependencies:

```powershell
pip install paho-mqtt scapy
```

---

## Wireshark (optional but recommended)

Wireshark helps verify generated traffic.

Install from:

https://www.wireshark.org/

Make sure **Npcap** is installed (included in Wireshark installer).

---

## Admin Privileges

The SSDP simulator sends raw Ethernet packets with scapy (`sendp`).

You **have to run the terminal as Administrator** when running `ssdp_sim.py`.

MQTT simulation **does NOT require Administrator privileges.**

# MQTT Simulation

The MQTT simulator mimics IoT devices periodically sending telemetry and occasional bursts of activity.

## Start MQTT Broker (recommended via WSL)

In WSL Ubuntu:

```bash
sudo apt update
sudo apt install -y mosquitto mosquitto-clients
sudo service mosquitto start
```

To view incoming messages:

```bash
mosquitto_sub -h 0.0.0.0 -t dams/sim/telemetry -v
```

Find the WSL IP address:

```bash
hostname -I
```

Use that IP as the `--host` param from Windows.

---

## Run MQTT Simulator

Passive telemetry:

```powershell
python mqtt_sim.py --host <BROKER_IP> --mode passive --seconds 60
```

Active burst:

```powershell
python mqtt_sim.py --host <BROKER_IP> --mode active --seconds 20
```

---

## Run MQTT Scenario

Simulates passive → active → passive behavior.

```powershell
powershell -ExecutionPolicy Bypass -File .\run_mqtt_scenarios.ps1 -Host <BROKER_IP>
```

# SSDP Discovery Simulation

The SSDP simulator generates **M-SEARCH discovery packets** to simulate IoT devices probing for LAN services.

Dest:

- Multicast IP: `239.255.255.250`
- UDP port: `1900`

---

## Find Network Interface

Run:

```powershell
python -c "from scapy.all import conf; print(conf.ifaces)"
```

Output may say something like:

```
Intel(R) Wi-Fi 6 AX201 160MHz
```

Use this name as the `--iface` arg.

---

## Run SSDP Simulator

Passive discovery chatter:

```powershell
python ssdp_sim.py --iface "<INTERFACE_NAME>" --mode passive --seconds 60 --passive-interval 30
```

Active burst:

```powershell
python ssdp_sim.py --iface "<INTERFACE_NAME>" --mode active --seconds 10 --active-burst-size 30
```

---

## Run SSDP Scenario

Passive → Active → Passive discovery traffic:

```powershell
powershell -ExecutionPolicy Bypass -File .\run_ssdp_scenarios.ps1 -Iface "<INTERFACE_NAME>"
```

Example:

```powershell
powershell -ExecutionPolicy Bypass -File .\run_ssdp_scenarios.ps1 -Iface "Intel(R) Wi-Fi 6 AX201 160MHz"
```

# Verifying Traffic in Wireshark

Start capturing on active interface (Wi-Fi or Ethernet).

### Filter only packets generated by SSDP simulator

```
ip.dst == 239.255.255.250 && udp.srcport == 1901 && udp.dstport == 1900
```
